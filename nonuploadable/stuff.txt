Video chat will not run on-chain. That's impossible and not how blockchain works.
Video chat is peer-to-peer:

WebRTC connects browsers directly
Video/audio streams go browser-to-browser via UDP
No blockchain involved in the actual call
No servers involved (except STUN/TURN for NAT traversal)

What blockchain does:

Authenticates users (wallet-based login)
Stores user profiles/identities
Maybe stores call metadata (who called whom, when, duration)
Handles permissions (who can call whom)

The video data itself:

Never touches the blockchain
Never touches your server (except signalling to start the call)
Goes directly between peers using WebRTC

Think of it like this:

Blockchain = authentication and access control
WebRTC = actual video streaming
Your WebSocket server = just introduces the peers to each other

This is the same model WhatsApp uses - their servers authenticate you and connect you to other users, but the actual call (if end-to-end encrypted) goes peer-to-peer.
The blockchain angle for Relay is:

Decentralised identity (not owned by a company)
Verifiable on-chain reputation/credentials
Censorship resistance
User owns their data

Not "video runs on blockchain" - that's technically nonsensical. The video runs peer-to-peer like all WebRTC apps, but the authentication and coordination layer is decentralised via Polkadot.
